# InsForge Integration Guide – StockSage AI

This guide explains how to connect your StockSage AI frontend to the InsForge backend for cloud-persistent storage and professional-grade data management.

## 1. Setup Database Schema
1. Log in to your **InsForge Console**.
2. Navigate to **SQL Editor**.
3. Copy the contents of [`insforge_schema.sql`](file:///C:/Users/lapto/.gemini/antigravity/scratch/stock-price-predictor/insforge_schema.sql).
4. Paste and click **Run**. This will create your tables (`profiles`, `portfolios`, `holdings`, `alerts`, `trades_history`) and enable Row-Level Security (RLS).

## 2. Environment Variables
Create a `.env` file in your root directory (or add to your existing configuration) with your project identifiers:

```env
VITE_INSFORGE_URL=https://your-project-id.insforge.ai
VITE_INSFORGE_ANON_KEY=your-anon-public-key
```

## 3. Client Initialization
Add an `insforge.js` file to your project to initialize the connection:

```javascript
/* ══════════════════════════════════════════════
   INSFORGE CLIENT – SDK Initialization
   ══════════════════════════════════════════════ */
import { createClient } from '@insforge/insforge-js';

const insforgeUrl = import.meta.env.VITE_INSFORGE_URL;
const insforgeKey = import.meta.env.VITE_INSFORGE_ANON_KEY;

export const insforge = createClient(insforgeUrl, insforgeKey);

/**
 * Example: Sync User Profile
 */
export async function syncUserProfile(user) {
    const { data, error } = await insforge
        .from('profiles')
        .upsert({
            id: user.id,
            display_name: user.name,
            updated_at: new Date(),
        });
    return { data, error };
}
```

## 4. Why use InsForge?
- **Cloud Persistence**: User portfolios are saved even if they clear browser cookies.
- **Security**: Data is protected by PostgreSQL RLS (Row Level Security).
- **Scalability**: Move from simulated data to a real production-grade backend with zero infrastructure management.
